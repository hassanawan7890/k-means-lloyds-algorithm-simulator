Documentation

How to Use

Four ways to load data, one simulation engine.

Manual Entry

Type coordinates directly into the Points and Centroids text boxes. One coordinate pair per line in x y format. Hit Apply to load into the scene.

Canvas Click

Switch to the Canvas tab and choose a tool - Points, Centroids, or Erase. Click anywhere on the simulation plane to place or remove entities.

A4 Import

Paste or upload a dataset in the original A4 format: point count N, then N coordinate pairs, then centroid count K, then K centroid pairs. Compatible with any file from generate_input.cpp.

N
x y

...
K
cx cy

Generate

Synthesize a random clustered dataset. Choose a profile (Balanced, Wide, or Insane), set point and centroid counts, adjust spread and noise, then hit Generate.

Running the Simulation

1

Load your scene using any of the four input methods above.

2

Switch to the Run tab. Set your stop mode - Epsilon halts when centroid movement falls below a threshold; Max Iterations halts after a fixed count.

3

Adjust Delay (animation speed) and Frames (smoothness of centroid movement).

4

Press Start. Watch points color by cluster and centroids migrate each iteration.

5

Use Pause to freeze mid-simulation, Reset to revert to the loaded scene, or Clear All to start fresh.

Tips

The Mirror Scene button copies the current canvas state back into the Manual text boxes - useful for saving mid-simulation positions.
The Shift stat shows the sum of squared centroid displacements per iteration. It reaches zero at full convergence.
The Voronoi-shaded background updates live - each region shows which centroid owns that area of the plane.
Set the seed in Generate mode to reproduce the same dataset exactly across runs.

Background

KMeans Simulator

The math and mechanics behind the clustering animation.

What is K-Means?

K-means clustering partitions a set of points into K groups (clusters) such that each point belongs to the cluster with the nearest centroid, and the sum of squared distances from each point to its assigned centroid (inertia) is minimized.

It's one of the most widely used unsupervised learning algorithms - efficient, interpretable, and easy to implement while still being provably NP-hard in general.

Lloyd-Forgy Method

This app implements the Lloyd-Forgy algorithm, the standard iterative refinement approach to k-means:

E-step
Assign - each point is assigned to the nearest centroid by Euclidean distance.
M-step
Update - each centroid moves to the mean position of all points assigned to it.
Check
Converge - repeat until centroid shift falls below epsilon, or max iterations is reached.

Convergence & Limitations

Lloyd's algorithm always converges - inertia is monotonically non-increasing - but it converges to a local minimum, not necessarily the global one. The final clustering depends on initialization.

Common failure modes: empty clusters (handled here by reinitializing from a random existing point), sensitivity to outliers, and poor performance when clusters are non-spherical or have very different densities.
